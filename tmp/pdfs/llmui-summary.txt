llmui - One-Page Summary

WHAT IT IS
A three-phase Android permission-scenario analysis pipeline that collects app UI traces, processes them into structured data, and performs scene, permission, and rule-based analysis. Supports LLM or vLLM backends and optional agent orchestration.

WHO IT IS FOR
Android security/privacy researchers or app analysts evaluating permission usage scenarios.

WHAT IT DOES
- Collects APK UI interaction traces using adb + fastbot (Phase1).
- Pulls and organizes raw fastbot outputs into a local data/raw store.
- Processes raw data with OCR and widget parsing to build permission chains (Phase2).
- Runs scene recognition via LLM or vLLM (Phase3 scene).
- Infers permissions and applies rule judgement; outputs structured JSON results (Phase3).
- Optionally performs LLM-based compliance analysis.
- Can orchestrate Phase3 steps via a LangGraph agent.

HOW IT WORKS (ARCHITECTURE)
APK(s) -> Phase1 (adb+fastbot) -> fastbot-* output in data/raw -> Phase2 (data_process.py) -> processed JSON/images in data/processed -> Phase3 (scene + permission + judge modules) -> analysis results JSON; optional LangGraph agent coordinates Phase3.

HOW TO RUN (MINIMAL)
- Install: python3 -m pip install -r requirements.txt
- Ensure: Android device connected via adb; vLLM or OpenAI-compatible endpoints available (set in env or settings.py).
- Run all phases: python3 src/main.py full /path/to/apk_or_dir
- Run only analysis: python3 src/main.py phase3 /path/to/processed_root --scene-mode text
