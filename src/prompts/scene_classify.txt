你需要进行安卓应用的功能场景分类任务。*functions*是安卓应用功能的映射表，*functions*中的每个key都是一个具体的场景类别，对应的value是描述该场景的关键字，只要UI符合关键字中的其中一个描述就能作为分类的依据。

**输入信息：**  
- `scenes`：该场景的 UI 列表，每个 UI 包含以下字段：  
  - `text`：UI 截图经过 OCR 处理后的文本内容。  
  - `widgets`：UI 中的组件列表，每个组件包含以下字段：  
    - `class`：组件的类型，例如 `Button`、`TextView`、`EditText` 等。  
    - `resource_id`：组件的资源 ID，可能暗示该组件的功能。  
    - `text`：组件的文本信息，可能进一步指示其功能。  
 {feature}

**任务描述**： 
 1. **分析 UI 场景**：根据 *scenes* 的 text 和 widgets 进行分类。
 2. **匹配分类**：在 *functions* 提供的类别列表中选择最符合的分类（最多三项）。
 3. **拓展定义**：如果现有类别不足以描述场景，可以给出新的定义。


 **JSON 输出格式**（请严格遵守，不要包含多余的文本）：
 ```json
 {
     \"analysis\": \"你的分析内容\",
     \"category\": [\"类别1\", \"类别2\", \"类别3\"],
     \"new_defined\": \"你的拓展定义（如果没有则填 null）\"
 }
 ```
 
 **规则**：
 - 确保 `category` 仅从 `functions` 里的 key 选择，最多 3 个。
 - `new_defined` 仅在 `category` 无法涵盖时填写，否则填 `null`。
 - 你的回答必须严格符合 JSON 格式，不要有额外的文本描述。
